	Java - portable - позволяет в отличии от c++ взаимодейстовать с api разных операционных систем
	JVM абстрогирует разработчика от операционной системы.
	Java - типо позволяет упростить разработку, ок, идея такая, чтобы снизить стоимость разработки.
	Java - типо безопасный, типо можно делать подписанный байт-код, ха-ха. В java невозможно напрямую адресовать память.
	JAAS(Java authentification ... standart) - позволяет сделать работу с аутентификацией, авторизацией и пр.
	Java - к упрощению относится автоматическая сборка мусора.
	java после версии 1.2 называется java2, она делится на 3 ветви: SE(Standart Edition) - ядро; EE(Enterprise Edition) - для упрощения использования в
большко количестве одинаковых операций - application server; ME(Micro Edition) - midlet(Старые приложения для телефонов), JavaCard - подмножество ME - приложения
для смарткарт.

	до версии 1.5 - ничего интересного
	начиная с версии 1.5 java начала называться Java5
	1.6 - ничего интересного
	1.7 - ничего интересного
	1.8 - добавили что - то интересное(добавили lambda - функциональность).
	1.9 - добавили модульность.
	Кто делает JVM: OracleJDK, openJDK(Это OracleJDK + 2% нативности), J9(от IBM), Apache Harmony, Azul.
	JVM + SysLib - минимальный набор для запуска java приложения - это называется JRE(Java Runtime Environment)
	javac - компилируется из исходника в java bite-код
	JVM + SysLib + tools = JDK
