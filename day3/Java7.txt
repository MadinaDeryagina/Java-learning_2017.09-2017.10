		Проблемы с типами.
	1. Переполнение.
	byte count = 0;
	for(int j = 0; j < 129; j++) {
		count++;
	}
	System.out.println(count);
	Что будет выведено? Ошибка, или -127 или может быть 0?
	Ответ: -127.
		Прикол в том, что произошло переполнение TypeOverflow. Самая большая проблема в том, что мы этого не узнаем.
		
	2. Потеря точности(Precition lost).
	System.out.println(1/3.); По идее должен вывести 0.333333333... Но у нас в плавающей точке конечное множество бит, т.е. по размеру оно ограничено.
	Для double будет 0.333333333
	Для float будет 0.33333334 
	Происходит потеря точности.
	
	System.out.println(.1 + .2); Как это сделать?  получается 0.300000000000000004 - это возникает из-за того, что мы 
оперируем отдельно разрядами, отдельно знаками. Есть сайт https://0.30000000000000004.com/ в котором можно посмотреть, как правильно складывать.
	
	Что делать с дробями? 
	1. способ округлять
	2.1 Сделать свой класс для дробей, поменять представления чисел, хранить отдельно числитель - отдельно знаменатель. 
	2.2 Создать свои обертки для корректных операций с дробями.
	
	3. Поведение с нулём.
	Для избежания деления на ноль, есть статические анализаторы: PMD, CheckStyle, FindBugs.
	1/0	 - ArythmeticException: divided by zero
	1/0. - в типе Double это будет PositiveInfinity
	0/0. - будет double.NaN
